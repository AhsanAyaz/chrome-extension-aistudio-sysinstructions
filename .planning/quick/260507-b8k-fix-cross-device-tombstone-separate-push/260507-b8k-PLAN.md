<objective>
Fix a cross-device tombstone bug where pulling remote items on device A causes those
items to be tombstoned when device A next pushes.

Root cause: `LAST_PUSHED_KEY` is written by both push flush AND pull update. The pull
writes ALL merged items (including device B's items) into `LAST_PUSHED_KEY` on device A.
When device A subsequently pushes its own (smaller) payload, `diffAndAccumulate` sees
device B's items in `lastPushed` but absent from the local payload → tombstones them.

Fix: Introduce a separate `PUSH_BASELINE_KEY` written ONLY by the push flush
(`writeLastPushed` in alarm-flush.ts). Use `pushBaseline` for tombstone eligibility in
`diffAndAccumulate`. Keep `LAST_PUSHED_KEY` for the loop-guard hash check (unchanged
dedup) — the pull engine continues writing to it as before.

Invariant after fix:
- `LAST_PUSHED_KEY` — written by both push flush AND pull update → loop guard only
- `PUSH_BASELINE_KEY` — written ONLY by push flush → tombstone eligibility baseline
- `pendingRegistry` — current burst accumulation → also tombstone-eligible
</objective>
