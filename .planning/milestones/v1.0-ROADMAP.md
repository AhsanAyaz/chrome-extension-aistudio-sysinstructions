## Overview

Full bidirectional sync of Google AI Studio system instructions across Chrome devices. The extension observes AI Studio's localStorage, assigns stable UUIDs to instructions, pushes diffs to chrome.storage.sync on a 30-second debounced alarm, and pulls remote changes back into the open AI Studio tab on any device. A Svelte 5 popup exposes sync status, the instruction list, Push Now / Pull Now manual controls, and JSON export/import. The toolbar badge reflects sync health (green/amber/red) with no silent error states.

## Milestone Summary

**Key Accomplishments:**
1. Storage schema locked with zero-migration `sysins:*` namespace, chunking, and tombstone semantics
2. MAIN-world localStorage observation pipeline with null-read guard and unknown-field passthrough
3. Push engine with UUID assignment, per-item diff, 30s debounced batched chrome.storage.sync writes
4. Pull engine with bidirectional sync, infinite-loop guard, multi-tab coordination, union-merge bootstrap
5. Account mismatch pre-flight using chrome.identity.getProfileUserInfo + AI Studio DOM selector
6. Svelte 5 popup with full sync visibility, Push/Pull controls, JSON export/import, 9-state error coverage

**Key Decisions:**
- Storage schema: `sysins:*` namespace, registry/body separation, 7 KB chunk budget — locked Phase 1
- UUID identity: crypto.randomUUID() on first sight; title-matching bootstrap-only
- Merge location: all merge logic in service worker; content script is relay only
- Chunking boundary: bufBytes + cpBytes > budget (strict >), Blob.size for byte-accurate measure
- applyRemote: max(updatedAt, deletedAt??0) authoritative timestamp; tie goes to tombstone
- importItems() separate from diffAndAccumulate() — import is additive, observe is full-replacement
- Svelte 5 onMount cleanup: synchronous return () => void; async work as void IIFE inside

**Issues Resolved:**
- CR-01: listener leak — onMount(async) returns Promise, not cleanup fn — fixed with sync onMount + void IIFE
- HR-01: destructive import — diffAndAccumulate tombstones absent items — fixed with importItems()
- HR-02: revokeObjectURL race — synchronous revoke before browser reads blob — fixed with setTimeout 1000ms
- HR-03: PULL_NOW no-op when registry absent — fixed with `?? {}` default
