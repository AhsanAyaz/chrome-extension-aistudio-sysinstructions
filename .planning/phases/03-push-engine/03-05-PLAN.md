  <how-to-verify>
    **Setup:** Load the unpacked extension from `.output/chrome-mv3/` in Chrome (chrome://extensions → Developer mode → Load unpacked). Open https://aistudio.google.com.

    **Verification Steps:**

    **SC-1: Edit lands in sync within 35 seconds**
    1. Open the AI Studio page and the Service Worker DevTools (chrome://extensions → find AI Studio Instructions Sync → Service Worker → Inspect)
    2. Edit any system instruction title or text in AI Studio and save
    3. In SW DevTools console: observe `[sysins] push: received N item(s)` log
    4. Wait up to 35 seconds (30s debounce + propagation)
    5. In SW DevTools console, run:
       ```javascript
       chrome.storage.sync.get(null).then(d => console.log(JSON.stringify(d, null, 2)))
       ```
    6. Confirm `sysins:registry` key exists with at least one UUID entry containing `updatedAt` > 0 and `deletedAt: null`
    7. Confirm at least one `sysins:body:<uuid>:c0` key exists with JSON text

    **SC-2: UUID stability — edit title, UUID unchanged**
    1. Note the UUID from the registry for one instruction
    2. Edit only the title of that instruction in AI Studio, save, wait 35s
    3. Run `chrome.storage.sync.get(null)` again
    4. Confirm the same UUID still exists (not replaced with a new one)
    5. Confirm `updatedAt` increased

    **SC-3: Chunking (instruction > 7 KB)**
    1. In SW DevTools console, create a large instruction manually:
       ```javascript
       localStorage.setItem('aistudio_all_system_instructions',
         JSON.stringify([{ title: 'Big', text: 'x'.repeat(10000) }]))
       // Trigger a synthetic StorageEvent to simulate AI Studio writing it:
       window.dispatchEvent(new StorageEvent('storage', {
         key: 'aistudio_all_system_instructions',
         newValue: JSON.stringify([{ title: 'Big', text: 'x'.repeat(10000) }])
       }))
       ```
    2. Wait 35 seconds
    3. In SW DevTools console, run:
       ```javascript
       chrome.storage.sync.get(null).then(d => {
         const keys = Object.keys(d);
         const bodyKeys = keys.filter(k => k.includes(':c'));
         console.log('body keys:', bodyKeys);
       })
       ```
    4. Confirm at least 2 body keys: `sysins:body:<uuid>:c0` AND `sysins:body:<uuid>:c1`

    **SC-4: Debounce — 5 rapid saves → 1 sync write**
    1. In DevTools Sources panel, add a breakpoint or spy on `chrome.storage.sync.set` — OR simply observe that after 5 rapid AI Studio saves (edit and save the same instruction 5 times within 10 seconds), only ONE chrome.storage.sync update is visible in the Application → Storage → Sync panel (the "Updated" count increments only once after 30s)
    2. Alternative: In SW DevTools console, run:
       ```javascript
       const orig = chrome.storage.sync.set.bind(chrome.storage.sync);
       let callCount = 0;
       chrome.storage.sync.set = (...args) => { callCount++; console.log('sync.set call #' + callCount); return orig(...args); };
       ```
       Then make 5 rapid edits in AI Studio. After 35s, callCount should be 1.

    **SC-5: Badge on failure**
    1. Simulate a quota error by temporarily monkey-patching in SW console:
       ```javascript
       const orig = chrome.storage.sync.set.bind(chrome.storage.sync);
       chrome.storage.sync.set = () => Promise.reject(new Error('QUOTA_BYTES exceeded'));
       ```
    2. Make an edit in AI Studio, wait 35 seconds
    3. Confirm the extension toolbar icon shows a red badge with '!'
    4. Restore: `chrome.storage.sync.set = orig`

    **Expected results for each SC:**
    - SC-1: sysins:registry present in sync, UUID + updatedAt + body key visible — PASS
    - SC-2: Same UUID after title rename — PASS
    - SC-3: Two or more c0/c1 body keys for 10 KB instruction — PASS
    - SC-4: Only 1 chrome.storage.sync.set call for 5 rapid edits — PASS
    - SC-5: Red badge appears within 5 seconds of failed flush — PASS
  </how-to-verify>
