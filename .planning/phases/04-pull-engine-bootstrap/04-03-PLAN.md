Create src/background/pull-engine.ts with:

1. Export function: handleRemoteChanged(changes, areaName) — async
   - Guard: if areaName !== 'sync' return
   - Guard: if REGISTRY_KEY not in changes return
   - Extract remoteRegistry = changes[REGISTRY_KEY].newValue as SyncRegistry
   - Guard: if remoteRegistry === undefined return
   - Call applyRemote(remoteRegistry) — merges remote into local registry
   - Call reconstructInstructions() — returns live Array<{uuid, title, text}>
   - Map to RawInstruction[] (keep uuid out of payload sent to CS, send only {title, text})
   - Call writeLastPushed on the merged payload (import exported writeLastPushed from alarm-flush.ts)
   - Call deliverToTab(mergedPayload)
   - Log: console.log('[sysins] pull-engine: applied', mergedPayload.length, 'item(s) from remote')

2. Internal function: deliverToTab(payload: RawInstruction[]) — async
   - chrome.tabs.query({ url: '*://aistudio.google.com/*', active: true })
   - If tab found (tab?.id !== undefined):
     - try chrome.tabs.sendMessage(tab.id, { type: 'APPLY_REMOTE', payload } satisfies ApplyRemoteMessage)
     - return on success
     - catch: fall through to pendingRemote path
   - If no tab OR sendMessage threw:
     - chrome.storage.local.set({ [PENDING_REMOTE_KEY]: { payload, enqueuedAt: Date.now() } satisfies PendingRemoteState })

Concrete lastPushed update in pull-engine:
```typescript
import { shortHash } from './hash';
import { LAST_PUSHED_KEY } from './sync-state';
import type { LastPushedSnapshot, LastPushedEntry } from '../shared/types';

// After reconstructInstructions(), build updated lastPushed snapshot:
const entries = await Promise.all(
  merged.map(async ({ uuid, title, text }) => {
    const bodyJson = JSON.stringify({ text });
    const [titleHash, bodyHash] = await Promise.all([shortHash(title), shortHash(bodyJson)]);
    const entry: LastPushedEntry = { titleHash, bodyHash, updatedAt: Date.now() };
    return [uuid, entry] as [string, LastPushedEntry];
  }),
);
const snapshot: LastPushedSnapshot = {};
for (const [uuid, entry] of entries) { snapshot[uuid] = entry; }
await chrome.storage.local.set({ [LAST_PUSHED_KEY]: snapshot });
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| chrome.storage.onChanged → pull engine | Remote registry data arrives via Chrome sync infrastructure — must validate shape before calling applyRemote |
| pull engine → content script | APPLY_REMOTE payload sent via tabs.sendMessage — CS receives only already-merged, tombstone-excluded data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | handleRemoteChanged remoteRegistry input | mitigate | Guard `remoteRegistry === undefined` before applyRemote; applyRemote() itself iterates Object.entries defensively; no additional validation needed for Phase 4 (V5 ASVS L1 met by existing applyRemote shape handling) |
| T-04-03-02 | Denial of Service | chrome.storage.onChanged loop amplification | mitigate | D-04: writeLastPushed after delivery ensures diffAndAccumulate returns hasChanges=false for the subsequent LS_CHANGED — no flush alarm scheduled. Guard in pull-engine (areaName + REGISTRY_KEY) prevents re-processing own push writes. |
| T-04-03-03 | Spoofing | APPLY_REMOTE to wrong tab | accept | chrome.tabs.sendMessage targets a specific tabId from tabs.query; only aistudio.google.com tabs are returned by the URL filter. No cross-origin delivery risk. |
| T-04-03-04 | Information Disclosure | Pull logs | mitigate | console.log emits only item count (integer) — never instruction titles or text. Consistent with T-03-02-b pattern from push-engine. |
| T-04-03-05 | Elevation of Privilege | pendingRemote written to chrome.storage.local | accept | sysins:local:* namespace is extension-private; only the extension's own content script reads it. No cross-extension access risk. |
