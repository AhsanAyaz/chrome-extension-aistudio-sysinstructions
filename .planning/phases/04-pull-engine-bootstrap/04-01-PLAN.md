Perform the following spike procedure manually in Chrome DevTools. This task cannot be automated — it requires a live browser with a signed-in Google account and an open AI Studio tab.

STEP 1 — Temporarily add identity.email to wxt.config.ts permissions:
  Change: permissions: ['storage', 'scripting', 'alarms'],
  To:     permissions: ['storage', 'scripting', 'alarms', 'identity.email'],
Then run: npm run build (or npx wxt build) to produce a fresh dist/.

STEP 2 — Load the unpacked extension in Chrome:
  Chrome → chrome://extensions → Load unpacked → select the dist/ output directory.
  (If already loaded, click the reload button.)

STEP 3 — Confirm chrome.identity.getProfileUserInfo works:
  Open Chrome DevTools → Application → Service Workers → click "inspect" on the extension SW.
  In the SW console, run:
    chrome.identity.getProfileUserInfo({ accountStatus: 'ANY' }, (info) => console.log(info));
  Record the output. Expected: { email: 'yourname@gmail.com', id: '...' }
  If email is empty string, note that too — it would mean an additional investigation step is needed.

STEP 4 — Find the AI Studio DOM selector:
  Open https://aistudio.google.com in a new tab.
  Open DevTools → Elements panel.
  Click the account avatar button in the top-right corner.
  In the DevTools Elements panel, inspect the element and look for:
    (a) An aria-label attribute containing the email (typical Google app pattern:
        "Google Account: Name (email@domain.com)")
    (b) A data-email or similar data attribute
    (c) Any element whose textContent directly contains the email
  Run in the console (AI Studio tab — NOT the SW inspector):
    document.querySelector('[aria-label*="Google Account"]')?.getAttribute('aria-label')
  Record the exact output. Also try:
    document.querySelector('[data-email]')?.dataset.email
    document.querySelector('a[href*="accounts.google.com"]')?.textContent
  Note which selector reliably returns the email string.

STEP 5 — Revert wxt.config.ts to the original permissions:
  Change back: permissions: ['storage', 'scripting', 'alarms'],
  This is a TEMPORARY spike — do NOT commit the identity.email permission yet.
  It will be added properly in plan 04-05.
