## Popup Dimensions and Layout

- Fixed popup width: 360px
- Height: auto, capped at 540px with internal scroll on instruction list
- Padding: 16px on all sides (popup shell)
- Popup sections (top-to-bottom):
  1. Status header — sync state, last-sync timestamp, instruction count
  2. Instruction list — scrollable, registry-derived, title + updatedAt per row
  3. Action row — Push Now and Pull Now buttons
  4. Export/Import row — Export and Import buttons
  5. Error/hint banner — conditionally rendered below action rows

Source: CONTEXT.md (D-01, D-02, D-03, D-05), REQUIREMENTS.md (UI-01, UI-02, UI-03, UI-04).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.5 |
| Label | 11px | 400 | 1.4 |
| Heading | 14px | 600 | 1.3 |

Notes:
- 13px body is appropriate for the constrained 360px Chrome popup viewport.
- Only two weights in use: 400 (regular) and 600 (semibold). No other weights.
- Instruction list rows use 13px body (400) for title, 11px label (400) for relative timestamp beneath.
- Section headings and sync-state label use 14px heading (600).

Badge colors: pre-established in Phase 3 codebase — amber #F59E0B for '!', red #EF4444 for '!', empty text for healthy. Not changed in Phase 5.

## Component States

### Instruction List Row

Each row renders:
- Line 1: instruction title (13px, weight 400, #111827) — truncated with ellipsis at 280px
- Line 2: relative timestamp (11px, weight 400, #6B7280) — format: "2 min ago" / "3 days ago" / "just now"

Timestamp display rule:
- < 60 seconds ago: "just now"
- 1–59 minutes ago: "{N} min ago"
- 1–23 hours ago: "{N} hr ago"
- 1–6 days ago: "{N} days ago"
- >= 7 days: ISO date string YYYY-MM-DD

Hover state: row background #F3F4F6 (no click action — list is display-only).
Empty list state: see Copywriting Contract.

Source: CONTEXT.md (discretion area — locale-relative chosen over ISO for readability in constrained width).

### Push Now / Pull Now Buttons

- Width: calc(50% - 4px) each, side-by-side in one row
- Height: 36px minimum
- Border-radius: 6px
- Push Now: filled, accent (#1A73E8 background, #FFFFFF text)
- Pull Now: outlined, secondary (#FFFFFF background, #111827 text, 1px solid #E5E7EB border)

Disabled state (during active sync): opacity 0.5, cursor not-allowed, pointer-events none.
Source: CONTEXT.md (D-04 fire-and-forget; D-05 reactive updates via onChanged).

### Export / Import Buttons

- Width: calc(50% - 4px) each, side-by-side in one row below Push/Pull
- Height: 36px minimum
- Border-radius: 6px
- Export: secondary style (#F3F4F6 background, #111827 text, 1px solid #E5E7EB)
- Import: filled accent style (#1A73E8 background, #FFFFFF text)
- Import button contains hidden `<input type="file" accept=".json">` — clicking Import programmatically triggers the file input

Source: CONTEXT.md (D-08 hidden file input, D-10 export schema).

### Status Header

- Sync state indicator: text label only — "Idle", "Syncing…", or "Error"
- lastSyncAt: relative timestamp with prefix "Last sync:" or "Never synced" if 0
- Instruction count: "{N} instructions" — 0 shows "No instructions yet"
- Layout: single-line flex row; state label on left, count on right; lastSyncAt below on second line

### Error / Hint Banner

Conditionally rendered at the bottom of the popup when `syncStatus.state === 'error'` or when a refresh hint is active.

- Error banner: 1px solid #EF4444 border, #FEF2F2 background tint, #111827 text, 8px padding
- Hint banner (PULL-03 refresh hint): 1px solid #F59E0B border, #FFFBEB background tint, #111827 text, dismissable via × button
- Banner width: 100% (full popup inner width)
- Dismiss button: 20×20px, positioned top-right of banner

Refresh hint text: "Pull applied — refresh AI Studio to see changes."
Refresh hint dismiss: clicking × sets a local reactive boolean; clears on next popup open.

Source: CONTEXT.md (specifics section — dismissable amber state), REQUIREMENTS.md (PULL-03).

## Export File Contract

- Filename: `aistudio-instructions-YYYY-MM-DD.json` (date = local date at time of export)
- Content: JSON array, prettified (2-space indent)
- Schema per item: `{ "title": string, "text": string, "uuid": string, "updatedAt": number }`
- Scope: live items only — where `deletedAt === null`
- Tombstones: excluded from export file

Source: CONTEXT.md (D-10, specifics section — 2-space indent).

---

## Import Validation Rules

Applied by popup before sending `IMPORT_ITEMS` message to SW:

1. File must parse as valid JSON array. On parse failure: show "Import failed: file is not valid JSON." No items sent.
2. Each item must have a non-empty string `title` and non-empty string `text`. Items failing this check are counted and reported. If ANY items fail: reject entire file with count message — no partial imports.
3. On full validation pass: send `IMPORT_ITEMS` message to SW with validated array. Show success message with count.

Source: CONTEXT.md (specifics section — reject missing title/text, show error in popup; D-09 routes through merge path).
