  <how-to-verify>
**Criterion 1: Popup status display (UI-01)**
1. Open Chrome with the extension loaded
2. Click the toolbar icon to open the popup
3. Confirm: popup shows last sync timestamp (or "Never synced"), sync state ("Idle" / "Syncing…" / "Error"), and instruction count
4. Expected: Status header visible with correct copy

**Criterion 2: Instruction list (UI-02)**
1. Add a system instruction in AI Studio (https://aistudio.google.com)
2. Wait up to 35 seconds for the push cycle to complete
3. Reopen popup
4. Confirm: instruction list shows the title and a relative timestamp ("just now" or "N min ago")
5. Expected: List reflects merged state — NOT raw localStorage

**Criterion 3: Push Now button (UI-03)**
1. Edit an instruction in AI Studio
2. Immediately click Push Now in the popup (without waiting 30 seconds)
3. In DevTools → Service Worker console, run: `chrome.storage.sync.get(null, console.log)`
4. Expected: The edited instruction appears in chrome.storage.sync within 5 seconds (not 30)

**Criterion 4: Pull Now button (UI-04)**
1. Click Pull Now in the popup
2. Confirm: popup shows the "Pull applied — refresh AI Studio to see changes." amber hint
3. If AI Studio is open, check whether the localStorage was updated (DevTools → Application → Local Storage)
4. Expected: Pull triggered; popup shows refresh hint; no infinite sync loop (no repeated sync.set calls)

**Criterion 5: Error and badge states (UI-05, UI-06)**
1. Simulate an account mismatch by checking the badge color after using a different Google account in AI Studio
2. OR: inspect chrome.storage.local in DevTools SW console: `chrome.storage.local.get('sysins:local:syncStatus', console.log)`
3. If errorState is set, confirm: popup banner shows the correct human-readable message from the Error State Copy table
4. Confirm: badge is empty (no text) when sync is healthy
5. Expected: no silent error state — every error is surfaced in the banner

**Criterion 5b: Export JSON (EXPORT-01)**
1. Click "Export JSON" in the popup
2. A file download should trigger: `aistudio-instructions-YYYY-MM-DD.json`
3. Open the file and confirm: JSON array with items having title, text, uuid, updatedAt fields
4. Confirm: no tombstoned items in the export (only live instructions)

**Criterion 5c: Import JSON (EXPORT-02)**
1. Take the exported JSON file, open it, add a new instruction manually: `{"title": "Test Import", "text": "imported text"}`
2. In the popup, click "Import JSON" and select the modified file
3. Confirm: popup shows "Imported N instruction(s). Syncing now."
4. Wait up to 35 seconds
5. Confirm: "Test Import" appears in AI Studio's instruction list OR in chrome.storage.sync registry
